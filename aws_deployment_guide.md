# 🚀 الدليل الشامل لرفع وتحديث النظام على سيرفر أمازون (AWS) 

هذا الدليل مصمم ليكون مرجعك الأساسي من لحظة استلامك لسيرفر جديد (Ubuntu) وحتى انطلاق مشروعك بشكل عام على الإنترنت (Public IP) وقبول التحديثات عبر Git.

---

## 🏗️ المرحلة الأولى: تهيئة السيرفر (تُنفذ لمرة واحدة فقط)

بمجرد دخولك وتأجير سيرفر **Ubuntu 22.04 LTS** على أمازون (EC2)، قم بالدخول إليه عبر SSH، ثم قم بتنفيذ الأوامر التالية بالترتيب لتثبيت الأدوات الأساسية (Git و Docker).

### 1️⃣ تحديث النظام وتثبيت أداة Git
```bash
sudo apt-get update && sudo apt-get upgrade -y
sudo apt-get install git -y
```

### 2️⃣ تثبيت Docker و Docker-Compose
```bash
# تنزيل سكريبت التثبيت الرسمي وتشغيله
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# تثبيت Docker-Compose
sudo apt-get install docker-compose-plugin -y
sudo apt-get install docker-compose -y

# إضافة المستخدم الحالي لقروب دوكر لكي لا نكتب sudo دائماً
sudo usermod -aG docker $USER
```
> [!NOTE]
> لتفعيل إضافة المستخدم לـ Docker، قم بتسجيل الخروج من הסيرفر (`exit`) ثم ادخل إليه مرة أخرى.

---

## 📥 المرحلة الثانية: سحب المشروع (Clone) وتكوين الـ IPs

الآن السيرفر جاهز لاستقبال شفرتك المصدرية من مستودعك الخاص (GitHub).

### 1️⃣ سحب المشروع عبر Git
افتح السيرفر المضيف ونفذ الأمر بمستودعك (مع استخدام الـ Token الشخصي لو كان مخفياً):
```bash
# ✅ صحيح
git clone https://github.com/osama841/drm-admin-panel.git

cd drm-admin-panel
```

### 2️⃣ ضبط ملفات البيئة (`.env`) والعناوين
الآن يجب تغيير الرابط الداخلي `http://localhost` في ملف الواجهة والباك إند ليستقبلا الإحصائيات من الـ Public IP الحقيقي أو الـ Domain.

* **لإعداد الباك إند:**
  قم بتعديل ملف `backend/.env` (أو إنشاءه لو لم يكن موجوداً، أو نسخه من `backend/.env.example`):
  ```bash
  nano backend/.env
  ```
  وتأكد من ضبط:
  ```env
  APP_ENV=production
  APP_DEBUG=false
  APP_URL=http://<IP_الخاص_بالباك_إند_أو_النطاق>
  ```

* **لإعداد الفرنت إند:**
  تأكد من تعديل الـ Base URL الموجه لـ Axios ليخاطب IP الباك إند أو (اسم الدومين). في مشروعك يقوم بجلبه غالباً إما من `VITE_API_URL` أو من `axios.js`.

## 🚀 المرحلة الثالثة: التشغيل الأول وصناعة الحاويات (Build & Run)

الآن، وبخطوة واحدة بفضل ملفات `docker-compose` والمجلدات الجاهزة:

### 1️⃣ بناء السيرفرات وإطلاقها (Build System)
العملية ستستغرق 1-3 دقائق يقوم فيها بتحميل NodeJS وبناء الـ React وتحميل PHP وتكوين خادمات Nginx:
```bash
docker-compose up -d --build
```

### 2️⃣ تثبيت الجداول والبيانات (Migration & Seeding)
بمجرد استقرار الحاويات (تتأكد من عملهم بكتابة `docker-compose ps`)، يجب أن تقوم بإخبار خادم الباك إند بتثبيت هيكلة قاعدة البيانات:
```bash
# الدخول إلى حاوية الباك إند وتنفيذ المايجريشن
docker-compose exec backend php artisan migrate --force

# (اختياري) إذا كنت تمتلك بذور بيانات أو حساب آدمن لتسجيل الدخول به
docker-compose exec backend php artisan db:seed --force

# صناعة المفاتيح الأساسية للارافيل
docker-compose exec backend php artisan key:generate
# ربط ملفات الصور والنظام
docker-compose exec backend php artisan storage:link
```

> [!SUCCESS]
> **مبروك!** مشروعك يعمل الآن رسمياً. 
> - **الواجهة:** تعمل على مسار بورت `80` (إذا كتبت الـ Public IP مباشرة في المتصفح ستظهر منصتك).
> - **باك إند (APIs):** يعمل على بورت `8000` (مثال `http://IP:8000/api`).

## 🔄 المرحلة الرابعة: إجراءت عمل التحديثات (Workflow for Updates)

هذا هو السيناريو الثابت في حال أنك قمت بتطوير ميزة جديدة أو إصلاح خلل في جهازك المحلي وقمت برفعه ל־GitHub (`git push`). كيف تجعل السيرفر يستقبل هذا التحديث؟

بشكل بسيط جداً ومختصر، تدخل مجلد المشروع في السيرفر وتكتب:

```bash
# 1. سحب التعديلات الجديدة المتواجدة في جيتهاب
git pull origin main

# 2. في حالة كان التعديل فقط في أكواد الواجهة (React) أو نصوص عادية
# دوكر كومبوز سيعيد تجميع الحاويات ويحدث الشفرة دون إيقاف السيرفر
docker-compose up -d --build

# 3. (أمر تنظيف إضافي 🧹) إن احتجت في يوم من الأيام لإغلاق كل الحاويات كلياً وفتحها
# docker-compose down
# docker-compose up -d --build
```

> [!TIP]
> بفضل طريقة كتابتنا لملف الدوكر (Volumes)، حينما تطفئ الحاويات وتقوم بالتحديثات، لن تفقد بيانات العُملاء ولا معلومات الجلسات التي خُزّنت في قاعدة بيانات حقل MySQL.
